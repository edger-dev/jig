# Task: dylint scaffold

Stand up dylint as a rule-enforcement mechanism — custom Rust lints for
type/AST-directed rules that clippy can't express and an architectural test
can't reach.

## Realizes

## Deferred (not blocked)

This is a deliberate YAGNI deferral, not an ambiguity block. No current rule
needs type/AST analysis — the one structural rule is a source-walking
architectural test. Building the scaffold now would add a fragile, nightly-pinned
`rustc_private` toolchain to a clean stable-only flake for zero present benefit.

**Trigger to pick this up:** the first rule that genuinely needs type or AST
resolution. The integration plan (nightly lint crate, opt-in `dylint = true` nix
wiring, a `cargo dylint` bacon job into `.bacon-claude-diagnostics`, `// rule:`
back-refs) is captured in the spec above.
