# Rule: every rule kino follows the schema

This is the meta-rule — a rule about rules. Any kino in the `rules` root MUST:

1. **Be named per the namespace convention** — `<owner>::<jig>::<local-slug>`,
   splitting on the last `::`. See kino://203f3ce17183ae7c3e0908db0591a089ed3cb69e3d8c28ca00666103347cf38e.
2. **Carry the ladder + mechanism metadata:**
   - `rule::ns` — the namespace (mirror of the name prefix)
   - `rule::rung` — current rung: `0` prose · `1` warn · `2` deny · `3` structural
   - `rule::target-rung` — the rung this rule is aiming for
   - `rule::mechanism` — `prose` · `clippy` · `rustc` · `arch-test` · `dylint`
   - `rule::category` — `safety` · `idiom` · `structure` · `meta`
   - `status::active` — `true` while the rule is in force
3. **Have a body in the rule shape:** a one-line *statement* of what is required,
   what is **rejected** and why, and the **escape hatch** for legitimate
   exceptions.

The ladder metadata is what makes graduation auditable
(kino://f3ea0cd6c5bff9a504656899415a86355de4846268c8d4d7568c22214e62e192) and
what lets the rung-0 hint digest be *generated* rather than hand-kept
(kino://4df64c1bac779eec6ac0806469abf38d31b92092eb4890319a03e96ac87b86ae).

**Rejected:** free-form rule kinos with rationale but no machine-readable rung.
Without `rule::rung` you cannot answer "which rules still cost the agent
context?" — the one query the whole framework is built to answer.

**Escape hatch:** none. A kino in `rules` that lacks this shape is malformed, not
a lax rule.
