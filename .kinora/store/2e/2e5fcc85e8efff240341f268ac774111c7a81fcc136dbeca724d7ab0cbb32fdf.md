# Plan: rules framework v1

Phases for building the jig lint-rules framework, tracked for the task
`jig::task::rules-framework-v1`.

## Phase 1 — Bootstrap jig's own ledger · DONE
Dogfood kinora in the jig repo: create `.kinora/config.styx` with a new `rules`
root beside `specs`/`tasks`/`inbox`. Rebuild the pinned kinora CLI (the vendored
binary was stale, missing `assign`/`commit`/`store --root`). Validated the full
store → commit → git → resolve round-trip against a custom root.

## Phase 2 — Framework specs · DONE
Four atomic specs authored and stored in `specs`: `rule-is-a-kino`,
`graduation-ladder`, `rule-namespace`, `rule-delivery`.

## Phase 3 — Seed rules · DONE
Stored in `rules`: `jig::meta::rule-schema` (the recursive meta-rule), and three
rust rules — `no-unwrap-in-lib`, `no-todo-committed` (idiom/clippy/rung-1),
`prefer-file-modules` (structure/arch-test/rung-1). Each carries the full
`rule::*` ladder metadata.

## Phase 4 — Enforcement in rust templates · IN PROGRESS
Wire the mechanism layer that ships to downstream projects:
- `templates/rust/Cargo.toml` — seed `[workspace.lints]` for the clippy rules,
  each line back-referenced `# rule: <name>`.
- `templates/rust/tests/architecture.rs` — the worked architectural-test example
  enforcing `prefer-file-modules`.
- `templates/rust/CLAUDE.md` — a "Lint Rules" section teaching the ladder, the
  diagnostics/test delivery split, and the `// implements:` back-ref.
- `templates/rust/rules/manifest.styx` — the downstream install manifest.

## Phase 5 — Commit · PENDING
`kinora commit` (fold roots) → `git commit` the `.kinora` ledger and the
template changes together.

## Out of scope (v1)
- The generated rung-0 hint digest (kept manual / documented for now).
- Full skill automation of the downstream installer (manifest + a `kinora store`
  loop is the seam; wiring it into `/jig init` comes next).
- A dylint scaffold — introduced only when a rule needs real AST/type analysis.
